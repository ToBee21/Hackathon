Stealth Intelligence Console — Architecture Brief

Before / After: Privacy Defense → Frontend Perception Agent

Porownanie tego, co jest faktycznie zbudowane w repo (Cloak & Dagger, read-only privacy defense) z architektura ze szkicu (Frontend Perception + Agent Loop). Diagramy z prawdziwych plikow: background.ts, content.ts.

01

BEFORE — co faktycznie jest w repo

Privacy-defense extension. Heurystyka classifyHeuristic tylko czyta (read-only risk scoring); reszta to ofensywne moduly obrony. Strumien jednokierunkowy: strona → ocena → storage → UI.

02

AFTER — szkic (Frontend Perception + Agent Loop)

Generyczny agent. Lokalny model (WebLLM / Gemini Nano) czyta strukture i dziala na stronie. Zamknieta petla: snapshot → reasoning → akcja → strona → nowy snapshot.

CONTROL LOOP — serce "after"

Jedna akcja, potem obserwacja. Bez planowania 10 klikniec naprzod — stan strony sie zmienia (modale, SPA, cookie bannery).
