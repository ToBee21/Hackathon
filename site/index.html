How it works

Three layers, all running inside your browser.

1. It reads the page like a profiler would

A local model scores each page for sensitive signals such as health, finance or identity context, and tells you when your visit could become a profiling event. The text never leaves the tab.

2. It blocks the network behind the page

A curated, license clean list of trackers, malware and phishing domains is enforced everywhere. On a page the AI marks sensitive, blocking escalates to a stricter tier scoped to that site only.

3. It tells you why

Every block is labelled with the reason and the source list, so you can see exactly which broker or malware host was about to receive your data.

Why you can trust it

Privacy software you cannot verify is just a promise. This one is built to be checked.

The AI never phones home

Models run in your browser over WebGPU and WebAssembly. There is no server to send your page text to, because there is no server.

Updates are signed, not trusted

The block list is delivered as data, never code, and every update carries an Ed25519 signature the extension verifies before it applies. A broken or hostile server cannot poison your install.

It fails safe

If an update is missing, stale or unsigned, the extension keeps the last good list. The worst a bad update can do is block too much, never unblock malware or leak data.

# every update is verified on your device fetch bundle.signed.json verify Ed25519 signature # reject if forged check version > last # reject rollback check size sane # reject anomalies apply block rules only # cannot unblock or redirect else keep last good list

Threat feeds are open source and license clean. Overlapping lists are deduplicated so the same bad domain is not counted twice.

Get the extension

A developer preview for Chromium based browsers. No sign up, no account, no email.

  1. Build the extension from source with the build script in the repository.
  2. Open your browser extensions page and turn on developer mode.
  3. Choose load unpacked and select the build output folder.

The package carries its full on device AI models, so it is large and ships as a build rather than a casual download. The live signed block list it pulls from is public and verifiable right now at /blocklist/manifest.json.
