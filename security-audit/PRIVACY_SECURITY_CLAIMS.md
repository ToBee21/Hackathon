# PRIVACY_SECURITY_CLAIMS

Audit rule: **a privacy/security feature without executable proof is treated as nonexistent for customer-trust claims.** This file classifies claims as verified, false, unimplemented, untested, misleading, or dependent on trust instead of enforcement.

## Claim classification

| Claim / implied promise | Classification | What is actually enforced | Evidence | Proof gate |
| --- | --- | --- | --- | --- |
| "Suwerenność danych" / active privacy protection | **Misleading if read broadly** | The extension has several bounded protections, but not a general privacy guarantee. Broad permissions, heuristics and browser/runtime trust remain. | `readme.md`, `SPECS.MD`, manifest in `package.json` | `tests/security/*.security.test.ts` |
| Local encryption with session-only key | **Dependent on trust instead of enforcement** | API tokens and aliases go through encrypted helpers. The "never on disk" part relies on Chrome `storage.session` semantics and memory fallback, not disk-forensics proof. | `src/shared/storage.ts`, `src/shared/emailAlias.ts`, `src/security/storageSchema.ts` | `storage-privacy.security.test.ts` |
| Generated email aliases are protected at rest | **Verified by tests and code evidence** | Current alias persistence uses `saveEncrypted(STORAGE_KEYS.EMAIL_ALIASES, ...)`; legacy plaintext aliases are read only for migration. | `src/shared/emailAlias.ts` | `storage-privacy.security.test.ts` |
| SimpleLogin is optional after user token | **Verified by tests and code evidence** | No install-time hardcoded `saveApiToken("simplelogin", ...)`; SimpleLogin endpoint remains explicit and user-action scoped. | `src/shared/emailAlias.ts`, `src/security/networkPolicy.ts`, `src/background.ts` | `no-hidden-network.security.test.ts`, `build-artifact-privacy.security.test.ts` |
| Panic Button clears cookies/cache/storage | **Verified by source tests, browser-dependent at runtime** | Background delegates to shared `panicButton()`, which calls `chrome.browsingData.remove`, clears extension local/session storage and resets the in-memory passphrase. Actual browser deletion success still depends on Chrome APIs. | `src/background.ts`, `src/shared/storage.ts` | `storage-privacy.security.test.ts` |
| AI Deep-Dive is local-only by default | **Verified for current runtime defaults** | Source/offscreen runtime set `allowRemoteModels = false`; proof command builds before scanning artifacts. This does not prove the bundled third-party runtime is incapable of network I/O under all future configs. | `src/shared/aiDeepDive/localNli.ts`, `assets/offscreen/offscreen.js`, built offscreen artifact | `no-hidden-network.security.test.ts`, `build-artifact-privacy.security.test.ts` |
| No raw AI/page text is retained in reports/logs/UI | **Verified for tested boundaries** | Stored risk results force `rawTextRetained: false`; raw LLM parse excerpts were removed; offscreen logs are allowlist-sanitized; floating UI no longer reads raw stream/json fields. Raw text still exists transiently during local classification. | `src/background/aiDeepDive/handleRiskResult.ts`, `src/shared/aiDeepDive/localLlm.ts`, `src/security/privacyGuards.ts`, `src/content/floatingWindow.ts`, `assets/offscreen/offscreen.js` | `no-exfiltration.security.test.ts`, `log-redaction.security.test.ts`, `page-dom-leakage.security.test.ts` |
| Model metadata proves locality | **Verified for tested sanitization only** | Trusted local model metadata is preserved only when incoming `model.localOnly` is true. Untrusted/remote metadata is dropped, not converted into fake local-only claims. | `src/background/aiDeepDive/handleRiskResult.ts` | `model-output-trust.security.test.ts` |
| Page content cannot command the model | **Verified for prompt contract and result parsing** | The LLM prompt frames page text as untrusted DATA; parsing clamps known fields and rejects raw output retention. This is not a formal prompt-injection proof. | `src/shared/aiDeepDive/localLlm.ts`, hostile fixture `prompt-injection.html` | `model-output-trust.security.test.ts`, existing AI tests |
| Sensitive pages are never scanned | **Misleading** | Sensitive-page behavior is heuristic. Tests/fixtures document dangerous cases, but there is no perfect exclusion guarantee. | `tests/fixtures/privacy-hostile/*`, `src/content/aiDeepDive/*` | `privacy-claims.security.test.ts`, `no-exfiltration.security.test.ts` |
| DataGhost anonymous decoy traffic protects identity | **Dependent on trust instead of enforcement** | Default silent network policy is enforced and noise is opt-in. Anonymity/profile poisoning effectiveness is not mathematically or externally proven. | `src/background.ts`, `src/security/networkPolicy.ts`, `src/shared/honeypot.ts`, `readme.md` | `no-hidden-network.security.test.ts` |
| Bionic Blur prevents fingerprinting/behavioral biometrics | **Untested as a security guarantee** | Source implements masking/jitter, but the proof suite does not measure real anti-fingerprinting resistance. README limitations are preserved. | `src/contents/bionic-blur-main.ts`, `readme.md` | partial: `page-dom-leakage.security.test.ts` only covers message leakage |
| Privacy Score is a security/privacy measurement | **Misleading** | It is a local UI heuristic over toggles/counters, not an independently measured protection score. | popup/dashboard code and README claim surface | `privacy-claims.security.test.ts` only locks claim presence/honesty clauses |
| Floating overlay hides data from host pages | **False** | Open Shadow DOM is explicitly documented as **not** a confidentiality boundary. It is safe only if secrets never enter it. | `src/content/floatingWindow.ts` | `page-dom-leakage.security.test.ts` |
| CND/offscreen messages are schema-validated | **Verified by tests and code evidence** | `isCndMessage()` delegates to an exact allowlist/schema validator; raw log fields crossing this boundary are rejected. | `src/security/validateMessage.ts`, `src/shared/messages.ts` | `message-boundary.security.test.ts` |
| Shipped build has no blanket `<all_urls>` host access | **Verified, but still broad** | Manifest avoids `<all_urls>` but still requests `http://*/*` and `https://*/*` plus powerful permissions. | built `manifest.json`, `package.json` | `permission-minimization.security.test.ts`, `build-artifact-privacy.security.test.ts` |
| No hidden third-party network traffic by default | **Verified for tested code paths** | Network policy is default silent; SimpleLogin/user-enabled noise endpoints are explicit; forbidden demo GA endpoint is not used as a real URL in self-tests. | `src/security/networkPolicy.ts`, `src/background.ts`, `src/shared/honeypot.ts` | `no-hidden-network.security.test.ts` |

## Bottom line

The project now has real proof gates for the most dangerous privacy claims, but the honest customer message must stay bounded: **local-by-default, explicit-network, redacted-boundaries, heuristic-not-magic**. Anything stronger is marketing bullshit unless new tests prove it.
